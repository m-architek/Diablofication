More HP on Level One

In non-modded game HP progression is divided into two parts: hit dice part and static modifier part. Here is example HP progression table for fighter (HPWAR.2DA):

SIDES ROLLS MODIFIER
1 10 1 0
2 10 1 0
3 10 1 0
4 10 1 0
5 10 1 0
6 10 1 0
7 10 1 0
8 10 1 0
9 10 1 0
10 10 0 3
11 10 0 3
12 10 0 3

This component calculates new HP progression table using following guidelines:

  • Character should receive all hit dice on level one.
  • On level two and up the same class depended static modifier should apply.
  • On last "hit dice" level amount of HP should be the same sa it would be without this component.

For example, the same table after component installation:

SIDES ROLLS MODIFIER
1 10 0 66
2 10 0 3
3 10 0 3

7 10 0 3
8 10 0 3
9 10 0 3
10 10 0 3

12 10 0 3

This component doesn't change the way constitution bonus works.

Companions HP is recalculated using the same rules. Due to some simplifications, amount of HP can be different for couple of points (usually in favor), especially for multi and dual class characters.
