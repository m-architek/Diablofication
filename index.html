Diablofication

Version: v0.5-SNAPSHOT (Download)
Game: BG1 EE v2.6 without SoD
Platform: Windows
Language: Universal (no new lines added)

Overview

Diablofication is Infinity Engine modification that aims to implement some of the mechanics known from Diablo games into Baldur's Gate games series.

Diablofication doesn't try to be total conversion mod. Instead, it tries to understand why certain mechanic is fun in Diablo and introduce some changes to Baldur's Gate to obtain similar result, without turning game upside down.

Consequently, Diablofication is collection of tweaks that focus on following aspects of the game:

  • Improvements for “no reload” gameplay, including ones that make this style of playing available for less experienced players.

  • Randomization and diversification of game content.

  • Various hack'n'slash flavored changes, especially focused on solo playthrough.

See components list for more details about mod content.

Diablofication tries to maintain compatibility with other mods by following best modding practices. Also, it tries to provide maximum effect at minimum cost by avoiding risky changes when it is not justified. See compatibility section for more details.

Installation

Unpack mod archive in main game directory. Run setup-Diablofication.exe and follow instructions on the screen. For deinstallation reverse steps of installation process. Run setup-Diablofication.exe and choose [U]ninstall option. After process finish, you can remove mod files from your game directory.

See here for more details about installing and playing Inifnity Engine modifications.

Some of mod components requires EEex to show their full potential. For getting access to all those additional features it is advised to install EEex before Diablofication. Diablofication was tested with EEex version v0.9.19-alpha, please use this version of EEex or later.

Some of mod components require starting new game to work properly. This also means that after mod deinstallation, using saves made when those components was yet installed can lead to unexpected issues. For details, see descriptions of particular component.

Compatibility

Diablofication is using WeiDU for patching game files in a way that will be compatible with patches done by other WeiDU based modifications. However, this “technical” compatibility not guarantee flawless in-game experience so please read following notes about cross-mod compatibility and installation order.

Diablofication is tweak collection mod and should be installed possibly late in mod installation process (preferably as the last mod). It tries to detect changes done by other modifications and act accordingly to provide best possible compatibility.

Each Diablofication component makes changes in different aspect of the game, detail compatibility notes are maintained per each component individually, please read them before installation.

Components

Diablofication mod consists of following components. All components are optional and can be installed independent from each other.

  • Character Respawn (Beta)
  • Remove Non-Combat Experience
  • More HP on Level One
  • Camera Lock (Beta)

Character Respawn (Beta)

In case of protagonist death, entire party will be resurrected in nearest temple (or in other sensible place, if temple is currently unavailable). This will however has consequence in certain amount of gold and experience points loss.

Motivation: Allowing for "no reload" playthrough in more forgivable manner.

Installation: This component should be installed after every mod that add new areas, items or spells. Starting new game is required for this component to work properly.

Compatibility: Resolving respawn place cannot work perfect in mod-added areas, it is advised to visit closest non-mod area before traveling there. Sometimes this component can produce strange results together with mods that add complex plot (e.g. when you die in mod-added area that is no longer reachable from nearest temple, what makes impossible to finish your quest). In case of such problems, you can always fallback to standard “reload” approach for this part of the game. In general, it is still advised to save game once in a while, and just not reload it, unless some unexpected problem occurs. This component is not compatible with mods that make changes related to protagonist death like “Death Cam” component of Tweaks Anthology or Protagonist Can Die tweak.

Tip: Protagonist doesn't drop equipment in the moment of death. However, enemies and party members still do. In case that battlefield is inaccessible or far away from respawn point, it can result with loot or equipment lose.

See here for more details (this can include spoilers).


Remove Non-Combat Experience

Removes experience points rewards from quests, dialogues, thief ability usages and spell learning.

Motivation: Removal of "free" XP in early parts of the game. Reducing overall XP amount for solo playthroughs. Increased freedom in picking dialogue options for powergamers.

Installation: This component should be installed after every mod that adds new quests and dialogues. Starting new game is not required.

Compatibility: This component will not be compatible with every modification that alters XP rewards, like XP adjustments components from EET Tweaks (excluding those related to combat XP).

See here for more details (this can include spoilers).


More HP on Level One

Significantly increases HP amount that protagonist and all companions receives on level one. At the same time, amount of HP gained with every level is decreased, so difference between HP from original rules and those applied by this component is getting smaller and smaller with every level up.

Motivation: Adding more hack'n'slash flavor to early portion of Baldur's Gate.

Installation: This component should be installed after every mod that adds new companions. Starting new game is required for this component to work properly.

Compatibility: This component will not be compatible with every modification that alters HP progression rules or HP of companions, like "Max HP at Level One", "Higher HP on Level Up", "Allow HP Rolls Through Level 20 [Angel]", "Maximum HP Creatures [the bigg]" (in "all" or "party-joinable NPCs" variants) from Tweaks Anthology.

Tip: This component ignores "Maximum HP" game option. It always calculates HP using "Maximum HP" rule.

See here for more details.


Camera Lock (Beta)

Double click on character portrait locks camera, making it move automatically with character movement, keeping character always in the center of the screen. You can unlock camera using right click on "select all" button.

Motivation: Moving camera automatically together with character movement saves tedious view scrolling, especially when playing solo. It also brings additional immersion by allowing you to experience game more from your character perspective.

Installation: This component should be installed after UI overhaul mods and after every mod that add new cutscenes. Starting new game is not required.

Compatibility: This component most likely wouldn't be compatible with other mods that change game camera behavior.

Limitations: If EEex not installed, camera lock is performed using character action list. That means, if character is in the middle of some action, like movement or spell casting, camera locked will be delayed until action finish. EEex allows for overcome this limitation. Additionally, it allows to perform camera lock on party leader after double click on "select all" button.

Tip: Move your character using right click button to avoid group select when view is still in move.

Tip: View movement is much smoother on higher framerate. You can adjust framerate by editing "Maximum Frame Rate" option in Baldur.lua config file (you can find it next to your save games). Note that this setting changes overall game speed.

See here for more details.
