Diablofication

Version: v0.6-SNAPSHOT (Download)
Game: BG1 EE v2.6 without SoD
Platform: Windows
Language: English, Polish

Overview

Diablofication is Infinity Engine modification that aims to implement some of the mechanics known from Diablo games into Baldur's Gate Enhanced Edition.

Diablofication doesn't try to be total conversion mod. Instead, it tries to understand why certain mechanic is fun in Diablo and introduce some changes to obtain similar result, without turning game upside down.

Consequently, Diablofication is collection of components that focus on following aspects of the game:

  • Improvements for “no reload” gameplay, including ones that make this style of playing available for less experienced players.

  • Randomization and diversification of game content.

  • Various hack'n'slash flavored changes, especially focused on increasing game dynamics.

See components list for more details about mod content.

Installation

Unpack mod archive in main game directory. Run setup-Diablofication.exe and follow instructions on the screen. For deinstallation reverse steps of installation process. Run setup-Diablofication.exe and choose [U]ninstall option. After process finish, you can remove mod files from your game directory.

See here for more details about installing and playing Inifnity Engine modifications.

Some of mod components requires EEex to show their full potential. For getting access to all those additional features it is advised to install EEex before Diablofication. Diablofication was tested with EEex version v0.9.19-alpha, please use this version of EEex or later.

Compatibility

Diablofication is using WeiDU for patching game files in a way that will be compatible with patches done by other WeiDU based modifications. However, this not guarantee flawless in-game experience in every setup, so if you plan to install Diablofication together with other mods, please read following notes about cross-mod compatibility and installation order.

Diablofication should be installed possibly late in mods installation process (preferably as the last mod). It tries to detect changes done by other modifications and act accordingly to provide best possible compatibility.

Each Diablofication component makes changes in different aspect of the game. Most common source of incompatibility could be to install Diablofication component together with other mod that changes same thing in different way. Detail compatibility notes are maintained per each component individually, please read them before installation.

Most of Diablofication components do not require starting new game to work properly and can be safely installed or uninstalled during playthrough. Exceptions from this behavior are stated in compatibility notes of particular components.

Components

Diablofication consists of several components. All components are optional and can be installed independent from each other. However, it is also possible to install them all, what allows to enjoy full Diablofication experiance.

  • Town Portal (Beta)
  • Identification For All
  • Character Respawn (Beta)
  • Remove Non-Combat Experience
  • Spells Regeneration (Beta)
  • More HP on Level One
  • Camera Lock (Beta)

Town Portal (Beta)

Town Portal is new level one arcane spell that teleports entire party to the nearest temple (or other safe place, if temple is unreachable). Character that casted Town Portal receives special ability, that can be used to teleport party back to place where spell was casted. Town Portal spell scrolls are added to store in every temple. Those scrolls can be used to cast Town Portal spell be every character that meet standard requirements for using scrolls (regardless of ability to use arcane magic).

Compatibility: This component should be installed after every mod that add new areas. Besides that there is no other compatibility restriction.

See here for more details (this can include spoilers).


Identification For All

This component changes Identify spell scrolls so they can be used be every character that meet standard requirements for using scrolls (regardless of ability to use arcane magic). Also, Identify scrolls are added to store in every temple.

Compatibility: This component has no compatibility restrictions.

See here for more details (this can include spoilers).


Character Respawn (Beta)

In case of protagonist death, entire party will be resurrected in the nearest temple (or other safe place, if temple is unreachable). This will however has consequence in certain amount of gold and experience points loss. After respawn, protagonist receives special ability, that can be used to teleport party back to the place where respawn was triggered.

Compatibility: This component should be installed after every mod that add new areas, items or spells. Deinstalltion of this component during playthrough is not recommended, it will leave your saved games in invalid state. This component is not compatible with other mods that make changes related to protagonist death like “Death Cam” component of Tweaks Anthology or Protagonist Can Die tweak.

See here for more details (this can include spoilers).


Remove Non-Combat Experience

Removes experience points rewards from quests, dialogues, thief ability usages and spells learning.

Compatibility: This component should be installed after every mod that adds new quests and dialogues. It is not compatible with every modification that alters XP rewards, like XP adjustments components from EET Tweaks (excluding those related to combat XP).

See here for more details (this can include spoilers).


Spells Regeneration (Beta)

Grants out of combat regeneration of one spell (arcane and divine) per round for all party members (including familiars).

Compatibility: This component has no compatibility restrictions.

See here for more details.


More HP on Level One

Significantly increases HP amount that protagonist and all companions receives on level one. At the same time, amount of HP gained with every level is decreased, so difference between HP from original rules and those applied by this component is getting smaller and smaller with every level up.

Compatibility: This component should be installed after every mod that adds new companions. Starting new game is required for this component to work properly. Also deinstalltion of this component during playthrough is not recommended, it will leave your saved games in invalid state. This component will not be compatible with every modification that alters HP progression rules or HP of companions, like "Max HP at Level One", "Higher HP on Level Up", "Allow HP Rolls Through Level 20 [Angel]", "Maximum HP Creatures [the bigg]" (in "all" or "party-joinable NPCs" variants) from Tweaks Anthology.

Tip: This component ignores "Maximum HP" game option. It always calculates HP using "Maximum HP" rule.

See here for more details.


Camera Lock (Beta)

Double click on character portrait locks camera, making it move automatically with character movement, keeping character always in the center of the screen. You can unlock camera using right click on "select all" button.

Compatibility: This component should be installed after UI overhaul mods and after every mod that add new cutscenes. Besides that there is no other compatibility restriction.

Limitations: If EEex not installed, camera lock is performed using character action list. That means, if character is in the middle of some action, like movement or spell casting, camera locked will be delayed until action finish. EEex allows for overcome this limitation. Additionally, it allows to perform camera lock on party leader after double click on "select all" button. In multiplayer game, camera unlock not work properly for players other then host, unless EEex is installed.

Tip: Move your character using right click button to avoid group select when view is still in move.

Tip: View movement is much smoother on higher framerate. You can adjust framerate by editing "Maximum Frame Rate" option in Baldur.lua config file (you can find it next to your save games). Note that this setting changes overall game speed.

See here for more details.
